# 성 방어 게임 - 기획 문서

## 1. 개요

| 항목 | 내용 |
|------|------|
| 장르 | 성 방어 / 서바이벌 |
| 플랫폼 | 웹 브라우저 (HTML5 Canvas) |
| 조작 | 마우스 (확장 시) |
| 플레이 시간 | 1회 5~10분 |
| 파일 구조 | `index.html` 단일 파일 |

## 2. 핵심 컨셉

> 맵 한가운데 있는 **성**을 사방에서 몰려오는 몬스터로부터 지켜내는 게임.
> 성은 자동으로 공격하며, 플레이어는 점점 강해지는 적의 파도를 얼마나 오래 버티는지 도전한다.

## 3. 게임 규칙

### 3.1 승패 조건
- **승리 조건**: 없음 (생존형 / 최고 점수 갱신)
- **패배 조건**: 성 HP가 0 이하가 되면 게임 오버

### 3.2 진행 흐름
1. 게임 시작 → 성이 맵 중앙에 배치
2. 화면 가장자리에서 몬스터가 스폰되어 성으로 직진
3. 성은 사거리 내 몬스터에게 자동으로 투사체 발사
4. 몬스터가 성에 도달하면 성 HP 감소 후 소멸 (혹은 지속 공격)
5. 시간이 지날수록 몬스터 스폰 속도/체력 증가
6. 성 HP 0 → 게임 오버 화면 → 재시작

## 4. 게임 요소

### 4.1 성 (Castle)
| 속성 | 초기값 | 비고 |
|------|--------|------|
| HP | 100 | 0이 되면 게임오버 |
| 공격력 | 10 | 투사체 1발 데미지 |
| 사거리 | 200px | 사거리 밖 적은 무시 |
| 공격 속도 | 1초 / 1발 | 쿨다운 |
| 타겟팅 | 가장 가까운 적 | 단일 타겟 |

### 4.2 몬스터 (Monster)
| 속성 | 초기값 | 비고 |
|------|--------|------|
| HP | 20 | 시간에 따라 증가 |
| 이동 속도 | 50px/s | 성 방향 직선 이동 |
| 공격력 | 10 | 성에 닿을 때 데미지 |
| 처치 보상 | 점수 +10 | (확장 시 골드) |

### 4.3 투사체 (Projectile)
- 성에서 발사, 타겟 몬스터를 추적하거나 직선 이동
- 몬스터 충돌 시 데미지 적용 후 소멸

## 5. 난이도 곡선

| 경과 시간 | 스폰 간격 | 몬스터 HP 배율 |
|-----------|-----------|----------------|
| 0~30초 | 2.0초 | 1.0배 |
| 30~60초 | 1.5초 | 1.2배 |
| 60~120초 | 1.0초 | 1.5배 |
| 120초~ | 0.5초 | 2.0배 이상 |

> 수치는 1차 MVP 기준이며, 플레이 테스트로 조정.

## 6. UI / 화면 구성

### 6.1 인게임 HUD
- **상단 좌측**: 성 HP 바 (현재/최대)
- **상단 우측**: 점수 / 경과 시간
- **중앙**: 성 + 사거리 표시 (옅은 원)
- **사방**: 이동 중인 몬스터, 발사된 투사체

### 6.2 게임 오버 화면
- "GAME OVER" 텍스트
- 최종 점수 / 생존 시간
- "다시 시작" 버튼

## 7. 개발 범위

### 7.1 MVP (1차)
- [x] 기획 문서 작성
- [ ] 캔버스 / 게임 루프
- [ ] 성 렌더링 + HP 바
- [ ] 몬스터 스폰 + 추적 이동
- [ ] 자동 공격 / 투사체 / 충돌 판정
- [ ] 점수 / 경과 시간 표시
- [ ] 게임 오버 + 재시작

### 7.2 확장 (진행 중 / 예정)
- **업그레이드 시스템** ([UPGRADE_SYSTEM.md](./UPGRADE_SYSTEM.md)) — 점수 임계값마다 카드 3장 중 1장 선택형 강화
- **보조 캐릭터(영웅)** ([HEROES.md](./HEROES.md)) — 카드로 영웅 소환/강화, 자동 공격
- 방어물(벽) + 경로탐색 (예정)
- 몬스터 종류 다양화 (빠른 적, 탱커, 원거리, 분열형)
- 보스 몬스터 (일정 시간마다 등장)
- 플레이어 능동 스킬 (광역 공격, 슬로우, 회복 등)
- 사운드 / 이펙트 / 파티클
- 최고 점수 저장 (localStorage)

## 8. 기술 스택

- **언어**: HTML / CSS / JavaScript (Vanilla)
- **렌더링**: HTML5 Canvas 2D
- **루프**: `requestAnimationFrame`
- **저장소**: 추후 `localStorage` (최고 점수)
- **외부 의존성**: 없음

## 9. 파일 구조

```
defense-game/
├── docs/
│   ├── GAME_DESIGN.md     ← 본 문서 (전체 게임 기획)
│   ├── UPGRADE_SYSTEM.md  ← 업그레이드 시스템 상세 기획
│   └── HEROES.md          ← 보조 캐릭터(영웅) 상세 기획
├── index.html             ← HTML 골격 + 스크립트 로드
├── style.css              ← 전체 스타일
└── js/
    ├── config.js          ← 상수/밸런스 (SPEED, 난이도 곡선 등)
    ├── heroes.js          ← 영웅 정의 + 공격 로직
    ├── cards.js           ← 카드 풀 + 등급 가중치 추첨
    ├── entities.js        ← 성/몬스터/투사체 생성·업데이트·렌더
    ├── ui.js              ← HUD/오버레이/카드 DOM 조작
    └── main.js            ← 게임 루프 + 초기화 + 입력
```
