# 보조 캐릭터(영웅) 시스템 기획서

## 1. 개요

성 주변에 자동 공격 유닛을 배치해 방어력을 보강하는 시스템.
영웅은 카드 시스템을 통해서만 획득/강화한다 (골드 미사용).

## 2. 영웅 종류

| 영웅 | 아이콘 | 색상 | 기본 ATK | CD | 사거리 | 공격 타입 | 비고 |
|------|--------|------|----------|-----|--------|-----------|------|
| 궁수 | 🏹 | `#7be88a` (녹) | 6 | 0.5초 | 180 | 단일 | 빠른 연사, DPS 12 |
| 마법사 | 🔮 | `#c264ff` (보) | 15 | 1.5초 | 200 | 광역 (반경 50) | 군집 처리, DPS 10 |
| 전사 | ⚔️ | `#ff9933` (주) | 25 | 1.0초 | 70 | 단일 | 근접 강타, DPS 25 |

## 3. 카드 시스템 통합

### 3.1 카드 풀
- 영웅 카드 3장이 기존 Rare 풀에 추가됨
- **항상 등장 후보** (이미 보유 중이어도 강화 카드로 등장)
- 등급: Rare (`RARE_RATE` 영향)

### 3.2 카드 동작
| 상태 | 카드 텍스트 | 효과 |
|------|------------|------|
| 미보유 | "🏹 궁수 소환" | 영웅 1체 등장, Lv.1 |
| 보유 중 | "🏹 궁수 강화 Lv.N → Lv.N+1" | ATK ×1.3, CD ×0.9 |

### 3.3 강화 누적
- 무한 누적 가능
- 강화당 ATK 30% 증가 (곱연산), CD 10% 감소 (곱연산)

## 4. 배치 (자동)

영웅은 성 중심에서 **60px** 떨어진 위치에 자동 배치:

| 순서 | 각도 | 위치 |
|------|------|------|
| 1번째 | 270° | 성의 위쪽 (12시) |
| 2번째 | 30°  | 성의 우하단 (4시) |
| 3번째 | 150° | 성의 좌하단 (8시) |

> 강화 시에도 위치 유지. 영웅끼리 위치 교환/재배치 없음.

## 5. 동작 규칙

### 5.1 공통
- 죽지 않음 (1차 단순화)
- 자동 공격: 사거리 내 가장 가까운 적
- 영웅의 사거리는 **영웅 위치 기준**, 성 사거리와 별개

### 5.2 마법사 광역
- 투사체가 첫 적에 명중 → 그 위치 반경 50px 내 모든 적에게 데미지
- 투사체 색상은 보라

## 6. 데이터 모델

### 6.1 영웅 객체
```js
{
  type: 'archer' | 'mage' | 'warrior',
  x, y, r: 12,
  baseAtk, atkMul: 1.0,    // 강화 누적
  baseCd, cdMul: 1.0,
  range,
  aoe: 0,                  // 0이면 단일, >0이면 광역 반경
  cdLeft: 0,
  level: 1,
  color: '#...',
}
```

### 6.2 게임 상태 추가
```js
state.heroes = []   // 보유 중인 영웅 배열
```

## 7. UI

### 7.1 좌측 하단 상태 패널
영웅 보유 시 별도 섹션 추가:
```
=== 영웅 ===
🏹 궁수 Lv.2 (DPS 15.6)
🔮 마법사 Lv.1 (DPS 10.0)
```

### 7.2 카드 화면
- 미보유: "소환" 표기
- 보유 중: "강화 Lv.N → Lv.N+1" 표기

## 8. 파일 구성

```
js/
├── heroes.js     ← 신규: 영웅 정의 + 생성/업데이트
└── cards.js      ← 영웅 카드 추가, apply 시그니처 변경(state 받음)
```

> `apply: castle => {...}` → `apply: state => {...}` 로 변경.
> 영웅 카드는 `state.heroes`에 접근해야 하므로.

## 9. 1차 구현 범위

- [ ] 영웅 데이터 모델 + 자동 배치 로직
- [ ] 영웅 자동 공격 + 투사체 발사 (단일/광역)
- [ ] 영웅 카드 3장 (소환/강화 동적 처리)
- [ ] 캔버스 렌더 (색상 원 + 사거리 옅게)
- [ ] 좌측 하단 영웅 정보 표시
- [ ] 카드 apply 시그니처 통일 (state)

## 10. 추후 확장

- 영웅 사망 / 부활 (보스 도입 시)
- 영웅 종류 추가 (성직자, 도적 등)
- 위치 수동 배치 (드래그)
- 영웅 전용 스킬 카드
