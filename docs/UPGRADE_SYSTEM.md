# 업그레이드 시스템 기획서

## 1. 개요

점수가 특정 임계값에 도달할 때마다 게임이 일시정지되고, 플레이어는 무작위로 제시된 **3장의 업그레이드 카드** 중 하나를 선택해 성을 강화한다.

- 목적: 자동 전투 게임에 **플레이어 의사결정** 추가
- 영감: 로그라이크형 빌드업 (Vampire Survivors / Brotato 류)

## 2. 트리거

### 2.1 임계값 공식
```
첫 임계값 = 50
다음 임계값 = 이전 임계값 + (50 × 현재 레벨)
```

| 레벨 | 누적 점수 | 추가 킬 |
|------|-----------|---------|
| 1 | 50 | 5킬 |
| 2 | 150 | +10킬 |
| 3 | 300 | +15킬 |
| 4 | 500 | +20킬 |
| 5 | 750 | +25킬 |
| 6 | 1,050 | +30킬 |
| 7 | 1,400 | +35킬 |

> 첫 카드는 5킬 내외(수 초)에 등장, 이후 완만하게 간격이 벌어짐.

### 2.2 동작
- 점수가 임계값 도달 시 → **게임 일시정지** (몬스터/투사체 멈춤)
- 카드 3장 오버레이 표시
- 카드 선택 시 효과 적용 → 게임 재개
- 다음 임계값으로 갱신

## 3. 카드 풀

### 3.1 등급 체계
| 등급 | 출현 확률 | 비고 |
|------|-----------|------|
| Common | 70% | 안정적 강화 |
| Rare | 30% | 강력하지만 희귀 |

각 카드 슬롯(3장)마다 독립적으로 등급 추첨 → 풀에서 랜덤 선택 (3장 중복 없음).

### 3.2 카드 목록

#### Common (70%)
| 이름 | 효과 | 비고 |
|------|------|------|
| 🗡 공격력 강화 | 성 공격력 **+5** | 누적 가능 |
| 🏹 사거리 확장 | 성 사거리 **+30** | 누적 가능 |
| ⚡ 공격 속도 | 공격 쿨다운 **-15%** (곱연산) | 누적 가능 |
| ❤️ 최대 체력 | 최대 HP **+30** + **현재 HP 풀 회복** | 누적 가능 |
| 💚 응급 치료 | 현재 HP **+40** (최대치 한도) | 즉시 효과 |
| 🌱 자동 재생 | 초당 HP **+1** | 누적 가능 |

#### Rare (30%)
| 이름 | 효과 | 비고 |
|------|------|------|
| 🎯 다중 발사 | 한 번에 발사하는 투사체 **+1**, 단 데미지 **×0.6** | 누적 가능 |
| 💥 강화 탄환 | 투사체 데미지 배율 **+30%** (공격력과 별도 곱) | 누적 가능 |
| 🛡 가시 방어 | 몬스터가 성 접촉 시 받은 데미지의 **100% 반사** | 누적 시 +50%p |

### 3.3 다중 발사 동작
- 사거리 내 **서로 다른 적**을 우선 타겟팅 (가까운 순)
- 적이 부족하면 같은 적에게 발사
- 데미지 페널티(×0.6)로 단일 강화 옵션 대비 균형 조정

## 4. UI / UX

### 4.1 카드 선택 화면
```
┌─────────────────────────────────────────────┐
│              LEVEL UP!                      │
│                                             │
│   ┌────────┐  ┌────────┐  ┌────────┐      │
│   │  🗡    │  │  🌱   │  │  🎯    │      │
│   │ 공격력 │  │ 재생   │  │다중발사│      │
│   │  +5    │  │ +1/s  │  │ +1발  │      │
│   │[Common]│  │[Common]│  │ [Rare] │      │
│   └────────┘  └────────┘  └────────┘      │
│                                             │
│       [1]        [2]        [3]            │
└─────────────────────────────────────────────┘
```

- **위치**: 화면 중앙
- **상호작용**: 마우스 클릭 + 키보드 1/2/3 단축키
- **시각적 강조**: Rare 카드는 테두리 색상 차별화 (예: 보라색)
- **호버**: 카드 확대 + 상세 설명 표시

### 4.2 게임 내 표시
- HUD에 **레벨** 표시 추가 (예: `Lv. 3`)
- 다음 임계값까지 남은 점수를 진행 바로 표시 (옵션)

## 5. 밸런스 조정 (기본 게임 변경)

업그레이드 시스템 도입에 따른 기본 밸런스 보정:

| 항목 | 기존 | 변경 |
|------|------|------|
| 후반 스폰 간격 | 0.5초 (120초~) | **0.4초 (한도)** |
| HP 스케일 (120초~) | 2.0배 | 그대로 |

> 이유: 업그레이드 누적이 후반 진행 속도를 따라잡기 어려움 → 스폰 한도 설정.

## 6. 데이터 모델

### 6.1 성(Castle) 추가 속성
```js
{
  // 기존
  hp, maxHp, atk, range, cd, cdLeft,

  // 추가
  level: 1,            // 현재 레벨
  cdMul: 1.0,          // 공격 쿨다운 배율 (낮을수록 빠름)
  multiShot: 1,        // 동시 발사 수
  dmgMul: 1.0,         // 데미지 배율
  thornsRatio: 0.0,    // 가시 반사 비율
  regen: 0.0,          // 초당 HP 회복량
}
```

### 6.2 게임 상태 추가
```js
{
  paused: false,           // 카드 선택 중 일시정지
  nextThreshold: 100,      // 다음 임계값
  cardChoices: [],         // 현재 표시 중인 카드 3장
}
```

### 6.3 카드 데이터 구조
```js
{
  id: 'atk_up',
  name: '공격력 강화',
  icon: '🗡',
  desc: '성 공격력 +5',
  rarity: 'common',        // 'common' | 'rare'
  apply: (castle) => { castle.atk += 5; }
}
```

## 8. 테스트 시나리오

| 케이스 | 기대 동작 |
|--------|-----------|
| 점수가 100 도달 | 게임 일시정지 + 카드 3장 표시 |
| 카드 클릭 | 효과 적용 + 게임 재개 + 다음 임계값 갱신 |
| 키보드 1/2/3 | 마우스 클릭과 동일 동작 |
| 카드 선택 중 몬스터 도달 | HP 깎이지 않음 (일시정지 상태) |
| 다중 발사 +1 (적 2마리) | 두 적에게 각각 1발씩 발사 |
| 다중 발사 +1 (적 1마리) | 같은 적에게 2발 발사 |
| 가시 방어 + 몬스터 접촉 | 성 HP -10, 몬스터 즉시 사망 (반사 10) |
| 자동 재생 (Lv1) + 시간 경과 | 초당 HP +1 (최대치 한도) |
| 게임 오버 후 재시작 | 모든 업그레이드 / 임계값 초기화 |
